AI-powered office suite
that runs on your machine

nOffice is a cross-platform desktop suite with five apps — nWrite, nSheet, nSlides, nImg, and nCode — all powered by a local Qwen3 8B model. No cloud, no telemetry, no subscription for the AI. Everything runs fully offline on your own hardware.

Apps

Five tools, one suite

Every app is a native Tauri window with its own AI sidebar. Launch from the nOffice shell, switch between them instantly.

W

nWrite

Full-featured word processor with TipTap/ProseMirror. Tables, images, math, track changes, footnotes, comments. Export to PDF, DOCX, HTML. Inline AI rewrite, summarize, and generate.

S

nSheet

Spreadsheet with HyperFormula engine and PixiJS WebGL virtualized grid. 1M+ rows. Full chart suite (bar, line, pie, scatter, area, bubble). AI formula generation and data analysis.

P

nSlides

Presentation tool with SVG + Canvas rendering. Full transitions including morph. Export to PDF and PPTX. AI slide generation and speaker note drafting.

I

nImg

Image editor with PixiJS WebGL canvas. Brush, eraser, shapes, text, clone stamp, gradient, blur. Full layer system. Generative AI via txt2img, img2img, inpainting, outpainting.

C

nCode

Multi-language IDE with Monaco Editor. Bundled LSP servers (pyright, clangd, rust-analyzer, gopls, solargraph). Git integration, multi-root workspaces. Ghost text, inline chat, explain, refactor.

AI

Every app has AI

Shared AI sidebar with 6 context modes: free chat, edit selection, explain, generate, summarize, translate. Local Qwen3 8B model. Fully offline inference.

AI

Local AI, built in

No cloud dependency. No API keys. No data leaving your machine. The Rust backend manages a llama.cpp instance running Qwen3 8B GGUF (Q4_K_M) with 32K context window. Chat and embedding models run concurrently.

▲

Qwen3 8B GGUF

Q4_K_M quantized, 32K context window. GPU acceleration via Metal, CUDA, ROCm, or Vulkan. Model stored at ~/.noffice/models/, auto-downloaded on first launch from HuggingFace.

◆

6 context modes

Free chat, edit selection, explain, generate, summarize, translate. The AI sidebar streams responses via Tauri IPC events bridging the llama.cpp SSE stream.

◼

Embedding pipeline

Background tokio::mpsc queue processes document embeddings via a separate qwen3-embedding model. Per-document sled KV store for vector indices.

⚙

Stable Diffusion

Auto1111 REST API integration at :7860. nImg supports txt2img, img2img, inpainting, outpainting. Rust probes port on launch, marks features as available or greyed-out.

✕

Zero telemetry

No analytics, no tracking, no prompt logging. The only network calls are the optional HuggingFace model download on first launch and periodic license check.

🔒

Fully offline

Once the model is downloaded, everything works without internet. No cloud dependency of any kind. Your documents, your data, your machine.

FAQ

Common questions

  • Qwen3 8B GGUF at Q4_K_M quantization with a 32K token context window. GPU acceleration via Metal (macOS), CUDA (NVIDIA), ROCm (AMD), or Vulkan (fallback).   • No. nOffice is proprietary/closed source. The source code is not publicly available.
